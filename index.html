B. Lógica de Negócios do Alerta (app.py)

A função analisar_risco_alagamento implementa a lógica principal do sistema, interpretando os dados para gerar um alerta de risco.

# Função para analisar o risco de alagamento
def analisar_risco_alagamento(dados_clima, dados_mare):
    chuva_mm = dados_clima.get('rain', {}).get('1h', 0)
    mare_atual = dados_mare['eventos'][0]  # Exemplo simplificado

    # Limiares para análise de risco
    LIMITE_CHUVA_MEDIA = 10  # mm
    LIMITE_CHUVA_ALTA = 25   # mm
    LIMITE_MARE_ALTA = 1.8 # metros

    # Lógica de decisão
    if chuva_mm > LIMITE_CHUVA_ALTA and mare_atual['altura_m'] > LIMITE_MARE_ALTA:
        return {"nivel": "Alto", "mensagem": "Risco de alagamento alto..."}
    elif chuva_mm > LIMITE_CHUVA_MEDIA:
        return {"nivel": "Médio", "mensagem": "Risco de alagamento médio..."}
    
    return {"nivel": "Baixo", "mensagem": "Condições normais, baixo risco..."}

2. Visualização de Dados: Tábua de Marés

Recurso Visual

Explicação da Tela: A aba "Tábua por Mês" renderiza um gráfico de linhas (via Chart.js) que exibe as alturas máximas e mínimas da maré para cada dia, permitindo uma fácil identificação de padrões e picos de maré.

Trecho de Código Relevante

Processamento de Dados da Maré (app.py)

A rota /tabua_mes lê o arquivo JSON local, processa os dados e os estrutura para que o frontend possa gerar o gráfico.

# Rota para obter dados de maré para a tábua do mês
@app.route('/tabua_mes')
def get_tabua_mes():
    cidade = request.args.get('cidade')
    # (...)
    cidade_encontrada = next((item for item in dados_mare_json if item["cidade"].lower() == cidade.lower()), None)
    # (...)
    # Processa os dados para extrair marés altas e baixas diárias
    dados_grafico = {
        "labels": [], "altas": [], "baixas": []
    }
    for dia_mare in cidade_encontrada['mares']:
        dados_grafico["labels"].append(dia_mare['data'])
        
        altas = [ev['altura_m'] for ev in dia_mare['eventos'] if ev['tipo'] == 'alta']
        baixas = [ev['altura_m'] for ev in dia_mare['eventos'] if ev['tipo'] == 'baixa']
        
        dados_grafico["altas"].append(max(altas) if altas else None)
        dados_grafico["baixas"].append(min(baixas) if baixas else None)

    return jsonify(dados_grafico)

Exemplo de Teste Unitário (test_app.py)

Este teste, usando pytest, simula uma chamada à rota /clima e valida se a resposta do servidor está correta, garantindo a estabilidade da API.

# Teste para a rota /clima com sucesso (usando mock)
def test_get_clima_success(client, mocker):
    # Mock da resposta da API externa para não depender da internet
    mock_response = mocker.Mock()
    mock_response.status_code = 200
    mock_response.json.return_value = {
        "name": "Recife",
        "main": {"temp": 27.0},
        "weather": [{"description": "céu limpo"}]
    }
    mocker.patch('requests.get', return_value=mock_response)
    
    # Executa a requisição de teste
    response = client.get('/clima?cidade=Recife')
    
    # Verifica as assertivas (se o resultado está correto)
    assert response.status_code == 200
    data = response.get_json()
    assert data['cidade'] == 'Recife'
    assert 'temperatura' in data
